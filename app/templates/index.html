GitHub sourcing

Search public repositories on GitHub, collect owners and top contributors, merge by username, score with three pillars (Builder, Product, Reach), then download a CSV. Requires GITHUB_TOKEN in .env.

How this run works

  • Builder — Each person gets a signal from how they appear on the repos we found (recency, damped contributor counts, optional public-event boost), then normalized with everyone else in this run.
  • Product — We score repos for product-like cues (description, homepage, topics, license; optionally README and CI workflows) and attach the strongest match to each person.
  • Reach — For up to the first N people (see Limits), we look up followers (log-scaled) and, if you set KOLs, overlap with follows; that feeds the third pillar before the weighted sum.

Scoring How important is each pillar?

Each person gets three raw signals (builder, product, reach). They are scaled 0–1 within this run, then combined: score = w_b×builder + w_p×product + w_r×reach. Enter any positive weights; the app normalizes them to sum to 1 (e.g. 3, 1, 1 → 60% / 20% / 20%).
